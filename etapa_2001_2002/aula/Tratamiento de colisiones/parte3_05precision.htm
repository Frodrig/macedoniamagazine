 PARTE 3: REACCIONES SIMPLES A LAS COLISIONES

Error de precisión

Los ordenadores traban con números finitos, lo que quiere decir que los números decimales tienen un máximo de precisión. En muchos casos la precisión proporcionada por éstos será suficiente; pero siempre habrá la posibilidad de que ciertas operaciones no den el resultado correcto debido a haber realizado la operación con números muy próximos al límite de la precisión soportada por los ordenadores.

 

  1. si ( a > 0 y b > 0 ) {
  2. c = a · b
  3. d = 1 / c
  4. }
Figura 4: Código que no funciona correctamente con valores muy pequeños

En la figura 4 se muestra un código aparentemente inofensivo. La comprobación de que a y b sean mayores que 0 implica que c no será nunca 0 y, por consiguiente, nunca habrá división por 0. Pero esto no es del todo cierto en un ordenador. Supongamos que a = 0.012 y b = 0.025 y que el máximo de precisión de un ordenador es 0.001. Esto nos da para a · b un resultado de 0.0003; pero como el ordenador solo puede guardar hasta 0.001 el valor se trunca para dar 0.000. En consecuencia c = 0 y la siguientes operaciones trabajan con una valor erróneo, que en este caso generará una división por cero.

 

Claro está que el ejemplo anterior es muy exagerado y los ordenadores disponen de una mayor precisión; pero el caso siempre se puede dar por buena que sea esta precisión. Un modo de solucionar esto sería modificar las comparaciones de la línea 1, de forma que no se compruebe si es mayor que 0, sino si es mayor que un valor épsilon muy cercano al cero, de modo que la operación de multiplicación no pueda dar en ningún caso 0 debido a una falta de precisión.

 

Existen evidentemente otros casos en que puede haber problemas por culpa de la falta de precisión, pero lo importante es entenderla para poder detectar estos casos y corregirlos.
